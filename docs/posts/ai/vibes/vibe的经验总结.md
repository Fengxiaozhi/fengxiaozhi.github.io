# vibe的经验总结
个人常用的几种Ai编程模式。

- tab+prompt（简单的逻辑处理使用tab+直接输入prompt处理）
- plan模式（复杂的功能开发使用plan模式）
- SDD驱动开发 （从0-1的功能开发模式）
- SubAgent模式 （待体验）
- MutilAgent模式（待体验）

## tab+prompt
这是最简单的AI 编程方式了，处理已有的工程代码，减少需要Ai重新理解整个工程的token，这其实并不完全是真正的vibe.这是最原始的Ai辅助编程，真正的vibe应该是其他几种模式也就是完整工程都有Ai理解掌握在它的上下文中但是这样需要消耗的token就更多。

### 场景
- 快速修复以及优化代码逻辑
- 实现简单的功能逻辑

## plan模式
当第一种tab + prpmpt的模式无法满足你的需求或者需要不断的和它对话才能得到你相对满意的结果时就应该去尝试plan模式了，它底层有个叫“苏格拉底”的思维模式。这种模式会不断深究你的需求和你探讨需求实现的关键功能点，直到这个讨论的目标接近你想要的结果了再开始去实现代码，这个过程最终产生一个vibe的plan给你去确认，你觉得OK了才开始去执行。当然这个模式所消耗的token比第一种模式大多了。

### 场景
- 实现小功能
- 快速实现

### 效果
使用plan模式得到效果能达到70%-80%，出来的结果还需要继续prompt完善。

## SDD驱动开发
SDD驱动开发全称Specification-Driven Development，就是在开始prompt之前先把你的需求在文档里写的清清楚楚，例如需求实现目标，实现需求需要使用的规范，接口定义规范，需求实现的边界判断，就是一份需求图纸。

SDD与非SDD的区别：
```
给我实现一个后台用户管理功能。
// 直接的prompt会让Ai去胡思乱想：我要使用vue 还是react,用户的字段随意定：名称手机号码邮箱？
```
使用SDD
```
// 实现一个后台用户管理功能
# 实现一个后台用户管理功能
目标是实现一个后台用户管理功能

## 技术要求
使用vue2.0 + element-ui

## 用户管理新增编辑页规范定义
## 接口规范定义

```
这样出来的效果可以说直接拉满，基本不需要再prompt调整。

### 场景
- 0-1的页面功能开发


### 效果
